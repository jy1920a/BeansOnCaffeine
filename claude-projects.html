Claude · Workspace feature

What are
Projects?

Self-contained workspaces in Claude for persistent context, scoped knowledge, and repeatable instructions.

Overview
Three things make a Project useful
Chat history
Past conversations keep the workstream grounded
Chats inside the Project retain context from earlier exchanges, so follow-up prompts stay tied to the same topic and workflow.
Project knowledge
Files and reference material live inside the workspace
Upload documents, text, code, and supporting files once, then let every chat in the Project draw on that same knowledge base.
Project instructions
Claude answers with a consistent role and style
Set tone, formatting, and response behavior for the entire Project so follow-up chats stay aligned without repeating the same briefing.
Feature 01 · Available to all users
Projects
A self-contained
workspace with chat
history, knowledge,
and instructions.
Persistent context
Chats stay connected to the same workstream, so you can continue analysis and drafting without rebuilding context each time.
Knowledge base
Add project-specific files and reference material so Claude can respond using the information that matters for that workspace.
Workspace instructions
Define how Claude should write, structure answers, or behave inside the Project so outputs stay consistent across chats.
Free-plan access
Projects are available to all users. Free accounts can create up to five Projects.
Feature 02 · Paid plans and Claude for Work
Scale and collaboration
Larger knowledge,
shared access,
and governed use
for teams.
Enhanced RAG on paid plans
On Pro, Max, Team, and Enterprise, Projects use Retrieval Augmented Generation when knowledge nears context limits, expanding capacity by up to 10×.
Shared Projects
Team and Enterprise users can share Projects with individuals, add people in bulk, or share across the organization when enabled.
Permission levels
Members can use a Project without editing it, or receive edit rights to contribute, manage members, and update instructions or knowledge.
Administrative control
Owners or Primary Owners can disable public Projects for the organization, removing organization-wide sharing for existing and future Projects.

Decision guide
Should this live in a Project?
Will this work need structured, reusable context across multiple interactions?
Do you need Claude to keep using the same files, notes, or code?
Do you need consistent instructions or shared access?
Projects
Use a Claude Project
This work benefits from reusable context, a shared knowledge base, repeatable instructions, or collaboration. A Project is the better fit.
Standard chat
A regular chat is enough
If the task is one-off, has little reusable context, and does not need ongoing files, instructions, or collaboration, you do not need a dedicated Project.
Bottom line. Projects = structured, reusable context. Standard chat = isolated, task-based interaction.
